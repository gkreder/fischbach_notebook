So going back into the Bokeh code for HSV it looks like the end result I need to plug into the rectangle plot is a pandas dataFrame that will associate a color to each entry. Each entry here will be an element of the PFAM grid.

I think I should make a data_prep function like I did for HSV but make it generalizable to different PFAM domains.

I already have the blocks_to_array funciton in the existing CAS data_prep code, I think I can work off of that.

I've set up the data_prep.py script in the top-level CAS folder that looks somewhat like the data_prep script from the HSV folder. A difference here is that the CAS data_prep script functions take a pfam target as an argument so that they can be called with various pfam key terms.

I just created a folder called scripts in the CAS top-directory and threw all the python scripts in there to make things cleaner. Also saw that if you want to import python functions/scripts from somewhere else (another folder) can do

from application.app.folder.file import func_name

as long as you have a file called init.py inside of that folder. init.py tells python that the directory can be treated like a package.

I've gotten the basic bokeh heatmap plot showing up, now just need to work on the color mapping
